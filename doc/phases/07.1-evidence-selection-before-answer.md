# Phase 7.1: Evidence Selection Before Final Answer

> **Status: Rejected**
> The prompt-only prototype (step 1 of the Recommended Validation Strategy) was implemented and tested empirically. It proved too slow in practice: the mandatory evidence-gathering + cite_sources turn before every factual answer introduced latency that made the UX unacceptable. This phase will not be pursued further.

## Motivation
Inline quote streaming reduced latency and works often enough to be useful, but it still has two structural weaknesses:
- citation metadata is tunneled through free-form model text
- the final answer can still drift away from the evidence that should support it

This phase explores a stricter flow where the model first gathers and selects evidence, and only then produces the final answer.

## Goals
- Keep citations as a first-class tool concern.
- Require tool usage whenever a factual answer depends on external evidence.
- Support citations for retrieved documents, user-provided facts, and other tool results.
- Capture the exact quoted text that the final answer is allowed to rely on.
- Make the final answer explicitly grounded in a preselected evidence set.

## Non-Goals
- Replace the current Phase 7 implementation immediately.
- Guarantee zero hallucinations.
- Redesign the UI in this phase.

## Proposed Two-Phase Flow

### 1. Evidence Gathering Phase
The model receives the user query and must decide whether factual grounding is required.

Rules:
- For purely social turns such as greetings, the model may answer directly.
- For any turn that requires facts, data, or attributed statements, the model must call tools first.
- Tool calls may include:
  - retrieval/search over indexed sources
  - lookup of relevant user-provided statements from conversation history
  - domain tools whose outputs contain factual data
- The model may iterate on tool calls until it has the required evidence.
- The phase terminates with a dedicated citation-selection tool call.

The new citation tool should accept a normalized list of evidence items across source types.

Conceptual evidence item shape:
- kind: search_result | user_statement | tool_result
- origin identifier: tool call id and/or message id
- exact quote text used as evidence
- source metadata required for later rendering

### 2. Final Answer Phase
The controller sends a second prompt that contains only the selected evidence set plus answer-writing instructions.

Rules:
- The final answer may only rely on the selected evidence.
- The answer should incorporate all selected evidence that is relevant to the user request.
- The answer should surface references in a deterministic way, ideally assigned by the controller/UI rather than invented by the model.

## Why This Could Be Better
- Evidence selection happens before final synthesis, not as a post-hoc justification step.
- One citation mechanism can cover retrieval results, user statements, and tool outputs.
- Exact quoted spans create a narrower grounding surface than chunk-level references alone.
- The final prompt can be made simpler because the evidence set is already curated.

## Main Risk
Even with a curated evidence set, the model can still hallucinate during final synthesis. This phase reduces the grounding gap, but does not eliminate it. Any implementation must therefore treat this as a stronger protocol, not as a correctness guarantee.

## Recommended Validation Strategy
- First test the concept with a prompt-only variation on top of the existing controller.
- Use the experiment to measure:
  - tool-call compliance
  - evidence selection quality
  - latency impact
  - answer quality versus current inline quote streaming
- Only introduce explicit controller phases if the prompt experiment shows clear upside.

## Acceptance Signal For Moving Beyond Prompt-Only Testing
- The model reliably calls tools for factual turns.
- The model reliably terminates evidence gathering with the citation-selection tool.
- Selected evidence is materially better than the current inline quote output.
- Latency remains acceptable.
- The final answer is measurably more trustworthy or easier to validate.
